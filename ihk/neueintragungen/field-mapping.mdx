## Complete Field Mapping

### Section 1: Nummer der Eintragung

**PDF Field:** Entry number (e.g., "HRB 123456")

**EVA Usage:**
- Used for verification only
- Confirms register type (HRB vs HRA)
- Not entered as separate field

<Note>
**HRB** = Handelsregister B (corporations: GmbH, UG, AG)
**HRA** = Handelsregister A (partnerships: OHG, KG, e.K.)
</Note>

---

### Section 2: Firma (Company Name)

**PDF Field:** "Firma: [Company Name]"

**EVA Fields:**

| EVA Field | Content | Rules |
|-----------|---------|-------|
| **Firmierung** | Full legal name | Copy EXACTLY from PDF, include legal form |
| **Kurzfirmierung** | Short name | Simple: same as full. Complex: abbreviated |
| **Alphasort** | Search keywords | 2+ words, no prohibited first words |
| **Schlagwort** | Additional keywords | Similar to Alphasort |

**Examples:**
```
PDF: "Holtkuh Verwaltungsgesellschaft mbH"
→ Firmierung: "Holtkuh Verwaltungsgesellschaft mbH"
→ Kurzfirmierung: "Holtkuh VerwGes mbH"
→ Alphasort: "Holtkuh Verwaltungsgesellschaft"
→ Schlagwort: "Holtkuh"
```

<Warning>
**Alphasort Prohibited First Words:**
- ❌ Verwaltungsgesellschaft
- ❌ Beteiligungsgesellschaft
- ❌ Schifffahrtsgesellschaft

Can use in Position 2+
</Warning>

---

### Section 2: Sitz (Address)

**PDF Field:** "Sitz: [Address]"

**EVA Fields:**

| EVA Field | Content | Hamburg Rules |
|-----------|---------|---------------|
| **Straße** | Street name | "Straße" → "Str." |
| **Hausnummer** | House number | Letter suffixes LOWERCASE: "12a" |
| **PLZ** | Postal code | Select from dropdown (validates) |
| **Ort** | City | Usually "Hamburg" |
| **Land** | Country | Usually "Deutschland" |
| **Zusätze** | Supplements | C/O, building names, floor |

**Examples:**
```
PDF: "Hauptstraße 12A, 20095 Hamburg"
→ Straße: "Hauptstr."
→ Hausnummer: "12a"
→ PLZ: "20095" (select from dropdown)
→ Ort: "Hamburg"
→ Land: "Deutschland"
```

---

### Section 2: Gegenstand (Business Purpose)

**PDF Field:** "Gegenstand des Unternehmens: [Description]"

**EVA Field:** **Branchenschlüssel** (Industry Codes)

**Mapping:**
1. Read business purpose description
2. Identify primary and secondary activities
3. Select 1-4 industry codes
4. Prioritize: Primary first, then secondary

**Examples:**

<Tabs>
  <Tab title="Example 1">
    **PDF:** "Betrieb einer Gaststätte und eines Hotels"

    **Codes:**
    - Restaurant operations (primary)
    - Hotel operations (secondary)
  </Tab>
  <Tab title="Example 2">
    **PDF:** "Vermögensverwaltung und Beteiligungen"

    **Codes:**
    - Asset management (primary)
    - Holdings/investments (secondary)
  </Tab>
  <Tab title="Example 3">
    **PDF:** "IT-Beratung und Softwareentwicklung"

    **Codes:**
    - IT consulting (primary)
    - Software development (secondary)
  </Tab>
</Tabs>

---

### Section 3: Stammkapital (Share Capital)

**PDF Field:** "Stammkapital: [Amount] EUR"

**EVA Field:** **Stammkapital** or **Kapital**

**Mapping:**
- Copy amount exactly
- German number format: 25.000,00 EUR
- Period for thousands, comma for decimals

**Common Amounts:**

| Legal Form | Minimum | Typical |
|------------|---------|---------|
| **GmbH** | 25.000,00 EUR | 25.000,00 EUR |
| **UG** | 1,00 EUR | 1,00 - 24.999,99 EUR |
| **AG** | 50.000,00 EUR | 50.000,00+ EUR |

<Note>
OHG, KG, e.K. may not have capital requirements or use different fields.
</Note>

---

### Section 4: Geschäftsführer (Directors)

**PDF Field:** "Geschäftsführer: [Name, DoB, Address]"

**EVA Section:** **Beteiligten** (multiple sub-fields per person)

#### Per Director Mapping:

<Steps>
  <Step title="Determine Function Code FIRST">
    Read "Allgemeine Vertretungsregelung"

    - **Code 24:** Acts alone (Alleinvertretung)
    - **Code 8:** Must act together (Gemeinsame Vertretung)
  </Step>

  <Step title="Search Person">
    Search by: Nachname + Vorname + Geburtsdatum

    If found → Assign existing. If not → Create new.
  </Step>

  <Step title="Enter Person Data">
    Fill all fields per person
  </Step>
</Steps>

#### Person Field Mapping:

| EVA Field | Source | Rules |
|-----------|--------|-------|
| **Nachname** | Last name from PDF | Include "von", "zu" as part of name |
| **Vorname** | First name(s) | **DEPENDS ON RESIDENCE!** |
| **Schlagwort** | Additional names | **German residents only** |
| **Geburtsdatum** | Birth date | DD.MM.YYYY format |
| **Geburtsort** | Birth place | If provided in PDF |
| **Wohnort** | Current residence | Determines name handling rules |
| **Funktion** | Function code | 24 or 8 (determined from Section 4) |
| **Beginndatum** | Start date | **Tag der Eintragung (Section 7)** |

#### Name Handling Rules (CRITICAL!):

<Tabs>
  <Tab title="German Residents">
    **If Wohnort = German city:**

    **Vorname:** FIRST given name ONLY

    **Schlagwort:** ALL additional given names

    **Example:**
    ```
    PDF: "Hans Peter Wilhelm Müller, wohnhaft in Hamburg"
    → Vorname: "Hans"
    → Schlagwort: "Peter Wilhelm"
    ```
  </Tab>
  <Tab title="Foreign Residents">
    **If Wohnort = Foreign city:**

    **Vorname:** ALL given names together

    **Schlagwort:** Empty or optional

    **Example:**
    ```
    PDF: "Hans Peter Wilhelm Müller, wohnhaft in Wien, Österreich"
    → Vorname: "Hans Peter Wilhelm"
    → Schlagwort: (empty)
    ```
  </Tab>
</Tabs>

---

### Section 6: Rechtsform (Legal Form)

**PDF Field:** Legal form text

**EVA Usage:**
- Determines entry type (Hauptsitz vs Zweigniederlassung vs GBR)
- Determines if capital required
- Affects navigation structure

**Mapping:**

| PDF Legal Form | EVA Entry Type | Notes |
|----------------|----------------|-------|
| GmbH, UG, AG | **Hauptsitz** | Standard corporations |
| OHG, KG, e.K. | **Hauptsitz** | Partnerships |
| eGbR | **GBR** | In HR Sonstige filter |
| Zweigniederlassung... | **Zweigniederlassung** | Branch office |

---

### Section 7: Tag der Eintragung (Registration Date)

**PDF Field:** "Tag der Eintragung: DD.MM.YYYY"

**EVA Usage:** **CRITICAL - Used everywhere!**

This date becomes:
- **Beginndatum** for ALL directors
- Start date for company operations
- Reference date throughout system

<Warning>
**Use the SAME date (Tag der Eintragung) for:**
- All Beteiligten (directors) Beginndatum
- Any other date fields requiring start date
- **NEVER use today's date or different dates per director!**
</Warning>

## Processing Workflow Mapping

### Step 1: Search (No Direct PDF Mapping)

**Action:** Search for existing entry BEFORE creating

**Fields to search:**
- Company name from Section 2
- Person names from Section 4 (optional but recommended)

**Why:** ~50% already exist as Betriebsstätte or Externer Hauptsitz

---

### Step 2: Company Data Entry

**PDF → EVA:**
- Section 2 Firma → Firmierung, Alphasort, Schlagwort
- Section 2 Sitz → Anschrift (all address fields)

---

### Step 3: Directors Entry

**PDF → EVA:**
- Section 4 "Allgemeine Vertretungsregelung" → Function Code (24 or 8)
- Section 4 Person details → Beteiligten (per person)
- Section 7 Tag der Eintragung → Beginndatum (all directors)

---

### Step 4: Industry Codes

**PDF → EVA:**
- Section 2 "Gegenstand" → Branchenschlüssel (1-4 codes)

---

### Step 5: Capital

**PDF → EVA:**
- Section 3 "Stammkapital" → Stammkapital field

---

## Special Cases & Exceptions

### Sitzverlegung (Relocation)

**PDF Indicator:** "verlegt von [Stadt] nach Hamburg"

**Action:** Search both old and new locations, convert existing entry

---

### Zweigniederlassung (Branch)

**PDF Indicator:** "Zweigniederlassung der [Parent Company]"

**Action:** Use Zweigniederlassung entry type, link to parent

---

### Multiple Directors

**PDF:** List of 2+ directors

**Action:** Enter each separately with SAME function code and Beginndatum
