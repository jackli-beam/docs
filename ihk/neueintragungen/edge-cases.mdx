## Phase 1: PDF Document Issues

### EC001: PDF Corruption or Unreadable
**Severity:** üî¥ CRITICAL

**Scenario:**
- PDF won't open or shows garbled text
- Extraction tool can't read the PDF
- Missing sections or error messages

**Resolution:**
1. Try alternative PDF viewer (Adobe, browser, system default)
2. Request file re-download and verify file size
3. Check Bundesanzeiger for better quality version
4. Request replacement from Amtsgericht Hamburg
5. Flag for IT support if system-wide

**Escalation:** Immediate - cannot process without readable document

---

### EC002: Multiple Companies in Single PDF
**Severity:** üü° WARNING

**Scenario:**
- PDF contains 2+ Neueintragungen
- Related companies filed together (e.g., GmbH & Co. KG)
- Same notary, bulk filing

**Resolution:**
1. Count distinct HRB/HRA numbers
2. Process each company as separate entry
3. Check for parent/subsidiary relationships
4. Document relationships in notes
5. Ensure all companies marked as processed

**Best Practice:** Process in logical order - parent before subsidiaries

  </Step>
  <Step title="Switch Mode">

**Prevention:**
- Never search single common words alone
- Always include 2-3 distinctive terms
- Start specific, broaden only if no results

### EC009: Typosquatting - Similar Names
**Severity:** üü° WARNING

**Scenario:**
- New company name extremely similar to existing one
- Examples: "M√ºller GmbH" vs "Mueller GmbH", "GlobalTech" vs "GlobalTeck"
- Same industry, nearby address

**Resolution:**
1. Verify exact spelling in PDF
2. Document existing similar entry (screenshot details)
3. Compare: Same directors? Same address? Same industry?
4. Assess legitimacy:
   - Different address + industry = probably fine
   - Same address + directors = likely related/legitimate
   - Same industry + similar to famous company = üö© FLAG

<Warning>
**Red Flags:**
- Similar name to DAX-listed company
- Famous brand with slight variation
- Umlaut substitution of well-known name
- Same industry + city + similar name = high risk
</Warning>

## Phase 3: Company Name Processing

### EC013: Name Exceeds Character Limit
**Severity:** üü° WARNING

**Scenario:**
- Company name very long, system field has limit
- Full name doesn't fit, truncation would lose meaning

**Resolution:**
1. Check exact character count
2. Try different fields (Firmierung vs Kurzfirmierung)
3. Abbreviation strategy (only with approval):
   - Abbreviate middle words, keep first and last
   - Keep legal form full (GmbH, not G.)
   - Example: "Hamburger Import Export Handelsgesellschaft mbH" ‚Üí "Hamburger Import Export GmbH"
4. Document full name in Schlagwort
5. Escalate for system fix if common

**Never abbreviate without approval!**

## Phase 4: Address Validation

### EC020: P.O. Box as Primary Address
**Severity:** üî¥ CRITICAL

**Scenario:**
- Address shows "Postfach" or "P.O. Box"
- No street address provided

<Warning>
**This is illegal!** P.O. Boxes CANNOT be used as official business address in Germany.
</Warning>

**Resolution:**
1. Verify PDF - is there a street address anywhere?
2. Look for "Gesch√§ftsanschrift" vs "Postanschrift"
3. If only P.O. Box:
   - **FLAG FOR SUPERVISOR IMMEDIATELY**
   - May be document error
   - Cannot proceed without physical address
4. Check Bundesanzeiger for street address

**Do NOT enter P.O. Box as official address!**

---

### EC021: Virtual Office Address
**Severity:** üü° WARNING

**Scenario:**
- Address is known virtual office provider (WeWork, Regus, Business Center)
- Multiple companies at exact same address

**Resolution:**
1. Virtual offices ARE legal in Germany if:
   - Real physical office space
   - Not just a postbox
   - Can receive mail
   - Possible to work there
2. Process normally - enter address exactly as shown
3. Note in internal comments if desired
4. Red flags (escalate if seen):
   - Obviously residential address
   - P.O. Box only
   - Suspicious pattern (100+ companies at tiny address)

---

### EC023: Address Not Yet in System
**Severity:** üü° WARNING

**Scenario:**
- Address validation fails, PLZ dropdown shows no match
- System: "Address nicht gefunden"
- Very new building or new street

**Resolution:**
1. Verify address is correct in PDF
2. External verification:
   - Google Maps: Does address exist?
   - Deutsche Post website: PLZ lookup
   - OpenStreetMap: Verify street
3. If address is real but not in system:
   - Try manual entry, override validation if system allows
   - Document: "New address, not in system database"
4. If system blocks: **FLAG FOR IT SUPPORT**, request address database update

### EC030: Mononym - Single Name Only
**Severity:** üü° WARNING

**Scenario:**
- Person has only one name (no surname OR no given name)
- Examples: Indonesian names ("Suharto"), single-word names
- Cultural naming conventions (Indonesia, Tibet, Myanmar)

**Resolution:**
1. Verify it's truly one name (check passport reference if mentioned)
2. System field allocation:
   - **Option A:** Put entire name in Nachname, Vorname blank/"-"
   - **Option B:** Put entire name in Vorname field
   - **Check internal guidelines** for standard
3. Mononyms typically foreign ‚Üí Use Foreign name rules
4. Document: "Person has mononym (single name)"

**Escalation:** Medium - need guidance on field allocation

### EC034: Gender Change / Name Change
**Severity:** üü° WARNING

**Scenario:**
- Person found with different name but same DoB + Geburtsort
- May be due to gender transition, marriage, legal name change

**Resolution:**
1. Verify identity (same DoB + Geburtsort = likely same person)
2. **Use NEW name from PDF** (current legal name)
3. System handling:
   - Create new entry OR update existing (check guidelines)
   - Note: "Name changed from [old name]"
4. **Privacy:** Gender transitions especially sensitive, minimal documentation
5. Assign to company using current name

<Note>
Germany's new self-determination law (Aug 2024) makes name/gender changes easier. Expect more cases.
</Note>

## Phase 6: Legal Representation

### EC041: No Matching Function Code
**Severity:** üü° WARNING

**Scenario:**
- Representation authority doesn't fit Code 24 or 8
- Unusual authority structure, hybrid or conditional rules

**Resolution:**
1. Try to map to closest:
   - Acts alone most of time ‚Üí Code 24
   - Usually needs another director ‚Üí Code 8
2. Check for "Other" option (Code 99 "Sonstige")
3. Document actual rules in notes/comments
4. **FLAG FOR SUPERVISOR** - may need new code created
5. Use same code for all directors, document decision

---

### EC043: Conditional Representation
**Severity:** üü° WARNING

**Scenario:**
- Representation authority is conditional
- Examples: "Director A for German matters, Director B for international"
- Subject matter limitations

**Resolution:**
1. Determine if internal vs external restriction
2. Map to function code (choose closest match)
3. **CRITICAL:** Document all conditions in notes
4. Copy verbatim from PDF
5. Flag if complex conditional structures

<Note>
German law allows limited representation authority if registered. Must be clearly documented in Handelsregister.
</Note>
